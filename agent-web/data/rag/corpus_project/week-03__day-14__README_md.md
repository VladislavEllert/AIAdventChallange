# День 14 — Инварианты и ограничения состояния

> ⭐ **Главный код задания:**
> - **[invariants/store.py](../../agent-cli/agent_cli/invariants/store.py)** — `load_invariants()`, `save_invariants()`, `add_invariant()` — YAML-хранилище отдельно от диалога (`data/invariants/default.yaml`).
> - **[invariants/checker.py](../../agent-cli/agent_cli/invariants/checker.py)** — `check_code()` (regex, без LLM) + `check_llm()` (LLM-арбитр проверяет готовый ответ).
> - **[core/prompt_builder.py](../../agent-cli/agent_cli/core/prompt_builder.py)** — инжект инвариантов в системный промпт секцией `[ИНВАРИАНТЫ] — нарушение ЗАПРЕЩЕНО`.
> - **[data/invariants/default.yaml](../../agent-cli/data/invariants/default.yaml)** — сами инварианты: запрет мата, запрет имени, стек Python only, запрет ORM.

> ▶ **Видео:** https://www.loom.com/share/216ee8cfa37f404087023ccd88672676

## Цель

Агент, который работает в рамках жёстких инвариантов — не может нарушить бизнес-правила или технические ограничения даже если пользователь явно просит.

## Архитектура

### Два слоя защиты

**Слой 1 — системный промпт** (превентивный):
```
[ИНВАРИАНТЫ] — нарушение ЗАПРЕЩЕНО, нет исключений
- Строгий запрет мата...
- Запрещено обращаться по имени...
- Стек только Python...
```
LLM видит до генерации → старается соблюдать.

**Слой 2 — код** (реактивный, жёсткий):
```python
ok, violation = check_llm(response, self.invariants, provider)
if not ok:
    self.agent.memory.pop_last_exchange()  # физический откат
    # пользователь видит: "Инвариант нарушен: ..."
```
Ответ сгенерирован → арбитр проверяет → при нарушении удаляет из памяти.

### Хранение

```yaml
# data/invariants/default.yaml
invariants:
  - Строгий запрет мата и нецензурной лексики
  - Запрещено обращаться к пользователю по имени
  - Стек только Python — запрещено предлагать JS/TS/PHP
  - Запрещено использовать сторонние ORM — только stdlib sqlite3
  - Бизнес-правило: запрещено давать советы по монетизации без запроса
```

Отдельный файл от диалога → можно менять не трогая историю.

## Что происходит при конфликте

1. Пользователь просит написать код на JavaScript
2. LLM пытается ответить (или отказывает сама — видела инвариант в промпте)
3. `check_llm()` → `НАРУШЕНИЕ: предложен JavaScript вместо Python`
4. `pop_last_exchange()` → ответ удалён из памяти
5. Пользователь видит: `[red]Инвариант нарушен: ...[/red]`

## Команды

```
/invariants list        — показать все инварианты
/invariants add <текст> — добавить новый инвариант
```
