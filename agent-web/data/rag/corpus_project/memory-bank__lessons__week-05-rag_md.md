# Неделя 5 — RAG (конспект лекции Гладкова, 2026-06-29)

> Источник видео: https://disk.yandex.ru/i/01KhOD_G7dBWPw
> Полный пересказ (summary+транскрипт) был приложен юзером; тут сжатая суть.

## Что такое RAG
**Retrieval Augmented Generation** — дополнение ответа LLM внешними знаниями **во время инференса**
(не вызов тула, а подгрузка данных в контекст).
- **MCP** = общение с *внешними* источниками (API/тулы). **RAG** = *внутренние* знания организации (вики, конфлюенс, доки).
- Аддитивно с fine-tuning и промпт-инжинирингом, не взаимоисключающе.

## Проблемы без RAG
Неполный/смешанный/большой контекст → галлюцинации. Худшее — **ложно-положительный** ответ
(похож на правду, но неверен). Хороший RAG-ответ = факт **+ ссылка на источник**.

## Эмбеддинги
- Эмбеддинг-модель: текст → **вектор** (массив чисел). Генеративная: текст → текст.
- **Косинусное сходство**: 1 = один смысл, 0 = про разное, −1 = противоположно.
- Размерность вектора: 768 / 1024 / 3072 — больше измерений = меньше коллизий (как в хэше).
- Пайплайн эмбеддинга: токенизация → embedding layer → self-attention → pooling (усреднение) → сравнение.

## Чанкинг
- **Фикс. размер**: режем каждые N токенов (напр. 500). Минус: рвём предложение → теряем контекст.
- **Семантический**: режем по блокам (абзацы/разделы/заголовки). Точнее, сложнее.
- **Overlap**: чанки перекрываются (напр. 500 с шагом 450) → границы «на замке».
- Размер чанка ~**500–1000 токенов**. Параметры подбираются **экспериментально**.

## Ollama (эмбеддинги локально) — рекомендация лекции
Приватно (данные не покидают комп), бесплатно, модели маленькие, совместимо с облачным эмбеддером,
удобно тестить RAG на малых объёмах. HTTP: `POST http://localhost:11434/api/embeddings`.
На VPS — заменить localhost на адрес.

## Реранкинг
Топ-K поиска может содержать нерелевантное (тема похожа, ответа нет).
**Reranking** = доп. переоценка найденных чанков **CrossEncoder**-моделью относительно запроса.
Медленнее (мс→сек), но точнее. Альтернативы для задания: **порог similarity / эвристика / query rewrite**.

## Метрики RAG
1. Качество ретривела (доля релевантных чанков, палец вверх/вниз).
2. Правдивость (отвечает из чанков, а не из общей памяти → есть ли ссылки).
3. Точность финала (сравнение с эталонными ответами, можно авто-регрессия).
Мини-анкета на вопрос: правильные чанки? ответ из контекста (ссылки)? финал верный?

## Когда RAG НЕ поможет
- Мало/плохие данные → fine-tuning.
- Сложная логика («если так, то так») → промпт-инжиниринг.
- Нужен стиль общения → агенты/тулинг.
- Устаревший формат → мультимодальный RAG / OCR.
RAG хранит **сырые векторы для семантического поиска**, поведение модели не меняет.

## Уточнения теории (Влад ↔ факты, проверять)
- Гладков говорит «параметров 768/3000» — это **размерность вектора** (dimensions), не число
  параметров модели. У nomic-embed-text ~137M параметров, но выход **768-мерный вектор**.
- «Нормализация = делить на максимум» — у него упрощённо. Для косинуса стандарт — **L2-норма**
  (делить на длину вектора √Σxᵢ²), тогда cos = скалярное произведение.
- Заголовок исходного md ошибочно «Неделя 2» — по расписанию курса RAG = **неделя 5**.
