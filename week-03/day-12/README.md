## Цель

Реализовать в TUI агенте **слоистый профиль пользователя** — статичный набор предпочтений (стиль ответов, запреты, стек технологий, интересы), который автоматически инжектится в каждый запрос. Доказать, что один агент с разными профилями дает **разные ответы на один и тот же вопрос**.

## Архитектура

### Модель профиля (5 слоёв)

```markdown
# Профиль: vladislav

## Профиль
Влад. Студент 1 курса ОмГТУ учусь на специальности программная инженерия углеказся ai-assisted разработкой

## Стиль
Хочу что бы кратко на ты и дружелюбно

## Правила/ограничения
строго запрещено материться врать и льстить

## Стек
claude code

## Увлечения
```

Каждый слой — отдельная секция `.md` файла. Инжектируется в `build_system_prompt()` как часть системного промпта.

### Персонализация на уровне промпта

1. **Создание профиля** → `/profile new [name]` → онбординг (4 вопроса) → `data/profiles/{name}.md`
2. **Переключение** → `/profile switch {name}` → агент сохраняет `self.current_profile`
3. **Инжект** → `Agent.respond()` вызывает `build_system_prompt(..., profile_content=profile.to_prompt_text())`
4. **Результат** → LLM видит `[Профиль пользователя]\n...` в системном промпте

### Демо в видео

1. Создаём 2 профиля с разными стилями.
2. Один вопрос → два разных ответа (тон, длина, формальность).
3. `/profile list` → таблица профилей.
4. `/profile show` → содержимое текущего.
