# iOS Agent App — милстоуны

Строим по шагам, не всё разом. Каждый милстоун = рабочее состояние, можно
остановиться и оно работает. Спека — `README.md`.

Принцип: сначала самый рисковый unknown (сеть + ключ + стриминг), потом обвес.

---

## M1 — Чат с одним агентом работает (рисковый фундамент)
**Цель:** доказать, что app ходит в ProxyAPI и стримит ответ.
- Xcode-проект, SwiftUI, iOS 17 target.
- `ProxyAPIClient`: POST `/v1/chat/completions` со `stream: true`, парсинг SSE.
- API-ключ: ввод в настройках → Keychain (НЕ хардкод, НЕ коммит).
- Один захардкоженный агент (Бро), один экран чата.
- Стриминговая печать ответа по токенам.
**Готово, когда:** написал сообщение → ответ печатается потоком.

## M2 — Несколько агентов + переключение
- Экран списка агентов (карточки).
- 2 пресета: Бро 🤝, Поисковик 🔎 (разные systemPrompt).
- Навигация: агент → его чат.
- SwiftData: сохраняем агентов и сообщения.
**Готово, когда:** переключаюсь между агентами, у каждого свой характер, история живёт после перезапуска.

## M3 — Чаты внутри агента + память чата
- У агента список чатов, «+ новый чат».
- Внутри чата полный контекст (короткая память), обрезка по бюджету токенов.
- Заголовки чатов (на старте — из первого сообщения).
**Готово, когда:** у агента несколько независимых чатов, каждый помнит свой диалог.

## M4 — Создание агентов пользователем
- Экран «создать агента»: имя, эмодзи, системный промпт.
- Удаление пользовательских (пресеты защищены `isBuiltIn`).
**Готово, когда:** создал своего агента с кастомным промптом, он работает.

## M5 — Крутилки параметров per-agent
- Настройки агента: temperature, top_p, top_k, seed, max_tokens.
- Параметры уходят в запрос; ловим 400 и показываем человеку (квирки моделей, см. README §7).
**Готово, когда:** меняю temperature у агента → вижу разницу в ответах вживую.

## M6 — Переключение моделей + дизайн-полиш + темы
- Выбор модели из массива `LLMModel` в настройках агента.
- Переключатель темы: светлая / тёмная / системная.
- Дизайн-проход: семантические цвета, SF Symbols, типографика, анимации.
**Готово, когда:** app выглядит «как от проф-дизайнера», тёмная тема полноценная, модели переключаются.

## M7 — Долгая память агента (memory bank между чатами)
- `AgentMemory.facts`: копилка между чатами.
- Наполнение: сначала кнопка «запомнить» вручную, потом авто-извлечение отдельным вызовом.
- Инжект фактов в system-контекст каждого чата агента.
- Потолок банка (лимит/сжатие).
**Готово, когда:** Бро в новом чате помнит факт из старого чата.

## M8 (Неделя 6) — бэкенд-прокси на VPS
- Сервер держит ключ ProxyAPI, app ходит к серверу.
- Ключ уходит с устройства совсем → app безопасно «на сервак».

---

## Порядок и риск

```
M1 (сеть+ключ+стрим) ← делать первым, тут все unknown
  → M2 (агенты) → M3 (чаты) → M4 (создание)
  → M5 (параметры) → M6 (модели+дизайн+темы)
  → M7 (долгая память)
  → M8 (бэкенд, Неделя 6)
```

M1–M4 = костяк. M5 завязан на материал Недели 1 (параметры). M7 = самая
интересная фича (двухуровневая память). Дизайн копим по ходу, финальный
полиш — M6.
