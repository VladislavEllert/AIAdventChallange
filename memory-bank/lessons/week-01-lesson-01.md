## Базовые термины (знать наизусть)

| Термин | Что это |
|---|---|
| **Код-ассистент** | Пишет код: Claude Code, Cursor, Copilot, Gemini, Codex, opencode. Интерфейс + модель. |
| **Агент** | ИИ, ограниченный контекстом под задачу. 6 стадий (см. ниже). |
| **LLM** | Большая языковая модель. **Сеть весов** (до ~1 трлн параметров), не «база векторов». Примеры: Claude Sonnet/Haiku/Opus. |
| **Контекстное окно** | Сколько токенов модель держит «в памяти» за раз. Переполнил → старое выпадает или суммаризируется. |
| **Токен** | Единица, на которую бьётся текст (вход и выход). |
| **Субагент** | **Разделить** контекст на изолированные окна под разные задачи (бэк/фронт/тесты). Цель — расширить рабочий контекст; экономия токенов — следствие. |
| **Скилл** | Мелкий навык агента (напр. поднять SQLite, дать команду). |
| **Рой (swarm)** | Контекст дробится на много мелких сущностей, один оркестратор дирижирует. Аналогия — рой дронов. |
| **Промптинг** | Всё есть текст. Субагенты и скиллы — тоже хитро уложенные промпты. |

## Техники промптинга (по нарастанию)

1. **Zero-shot** — строчка в чат. В современных моделях почти не используется в чистом виде.
2. **Chain of Thought** — просишь расписать ход рассуждений. Резко поднимает точность.
3. **Deliberate reasoning** — один вопрос с разных ролей/сторон («как физик, как химик…»). Хорошо комбинируется с CoT.
4. **Метапромтинг** — одна модель пишет промт для другой.
5. **Swarm-промтинг** — несколько моделей отвечают, потом сводишь воедино (сам или нейронкой).

## 6 стадий агента (агентинг)

Аналогия с 6-ступенчатой цепочкой нейронов в мозге.

1. **Perception** — вход (промт, данные, сенсоры).
2. **Memory** — долговременная память (файл/БД; переживает перезагрузку — у человека так нельзя).
3. **Reasoning** — способ размышления (CoT + deliberate).
4. **Planning** — рождается цель и план работ.
5. **Action** — действие (генерация кода, поиск). Что видишь — уже финал; главное прошло раньше.
6. **Feedback** — оценка «то / не то»; веса перестраиваются в рамках сессии.

## Параметры LLM (которые крутишь)

| Параметр | Что делает |
|---|---|
| **Temperature** (0–1) | 0 = сухие точные ответы, 1 = максимальный разброс / «бред». В части моделей шкала перевёрнута. |
| **Top-P** | Отсечение кандидатов по накопленной вероятности (порог). |
| **Top-K** | Отсечение по количеству (только N первых слов). В связке с Top-P. |
| **Max tokens** | На сколько токенов режется контекст. Больше ≠ лучше, просто дороже. |
| **Seed** | Зерно рандома; управляемая недетерминированность. |
| **Frequency / Presence penalty** | Мягкий штраф за повтор слов (чтоб не «дом-дом-дом»). |

**Правило:** точная задача (математика) → вниз. Творчество → вверх.

## Три типа промтов

- **System** — ядро сессии, стартовая настройка весов («ты программист на Kotlin с 20-летним стажем»). В коде — отдельный параметр в JSON-запросе. Переопределил → сессия с нуля.
- **User** — то, что пишешь ты.
- **Assistant** — то, что отвечает модель.

---

## 5 параметров, по которым отличаются модели

1. **Кол-во параметров** — до ~1 трлн; коммерческие модели динамически «обрезают» (квантизация) → экономия видеопамяти ценой ума.
2. **Inference speed** — скорость генерации токена.
3. **Context length** — заявляют до 10 млн, реально работает ~1 млн.
4. **Reasoning quality** — зависит от архитектуры.
5. **Экономика** — цена за 1 млн токенов.
