## Задача-ловушка

«Помыть машину, мойка в 50 м, ехать или идти?» Верно — **ехать** (машина = объект
мойки, её надо доставить). Человеку очевидно, слабой LLM (`gemini-2.5-flash-lite`) —
нет. Все 4 способа сперва отвечали «пешком».

---

## 4 способа рассуждения (техники из Урока 1, проверены вживую)

| | Способ | Техника из лекции |
|---|---|---|
| A | прямой вопрос, без инструкций | zero-shot |
| B | «решай пошагово» | chain-of-thought |
| C | модель сама пишет промпт, потом им решает | метапромтинг |
| D | группа экспертов (аналитик/инженер/критик) | deliberate reasoning / swarm |

Модель одна, `temperature=0` → разница **только от способа**, не от рандома.

---

## Вывод 1 — мультиагентность (эксперты) точнее всего

**D победил.** Группа экспертов дала верный ответ там, где остальные сели в лужу.
Почему: способ требует **прямого решения цели** (ИНЖЕНЕР: «чтобы помыть → доставить
машину → ехать»), потом другие роли **смотрят на результат и спорят** (КРИТИК ищет
дыры). Один проход легко уезжает в ложную метрику (топливо, экология); разбор
ролями ловит это.

**Главное:** агенты круче, когда **видят результат друг друга и работают вместе** —
один предлагает, другой критикует, третий синтезирует. Сильнее одного монолога
(CoT). CoT даёт ровные шаги, но при неверной посылке аккуратно обоснует
неправильный ответ (перл B: «ехать на грязной машине нелогично, опять запачкается»).

Порядок точности тут: **D (эксперты) > B (пошагово) > C (meta) ≈ A (прямой)**.

---

## Вывод 2 — промпт решает; намёки не работают, работает жёсткий факт

Уточняли по шагам: «свою машину» → «которую надо помыть» → «стоит у дома» — **три
намёка не сдвинули ответ**. Сдвинул только **жёсткий factual constraint** («мыть
можно только на мойке, дома негде»). Слабая модель цепляется за поверхностный якорь
(«50 м → близко → пешком»), намёки игнорит. Нужен не намёк, а ограничение, что
**меняет постановку задачи**.

---

## Вывод 3 — температура: точность ↔ разнообразие, а НЕ «тупее ↔ умнее»

Подняли `temperature` 0 → 1, ждали «станет креативнее, поймает ответ». Нет.
temperature=1 = **шум в выборе токенов**. Три прогона: ответы заскакали,
**сломался даже D** (на temp=0 железно верный) — лишний «второй раунд аналитика»
увёл синтез в «пешком».

- Высокая температура **не добавляет ума** — добавляет разброс. Может случайно
  попасть в верный ответ, может развалить рабочий способ.
- Задачи с **одним верным ответом** → низкая температура + структура рассуждения.
- Высокая температура полезна для **брейншторма/текстов**, где нужно разнообразие.

---

## Вывод 4 — self-consistency: как гасить шум, не теряя рандом

Если температура нужна >0, надёжность вытягивают **голосованием большинством**
(self-consistency, статья Google 2022). Это **не параметр API, а приём в коде**:
гоняешь один способ N раз при temp>0, берёшь самый частый ответ.

Прогон ×5 при temp=0.7: **D → «ехать» большинством** даже под шумом; A и C залипли
«пешком» 5/5; B скакал 3/2. Подтверждает Вывод 1: надёжность даёт **структура +
агрегация**, не подкрутка одной ручки.
