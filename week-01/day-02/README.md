# Неделя 1 — День 2

## Задание

Отправить один и тот же запрос с разным уровнем контроля ответа через API:
- добавить явное описание формата ответа;
- добавить ограничение на длину ответа;
- добавить условие завершения (stop sequence / явная инструкция).

Сравнить ответы: без ограничений vs с ограничениями.

## Формат сдачи

**Видео + код. Видео важнее кода.**

## Что внутри

- `compare.py` — вводишь вопрос в консоли, скрипт шлёт его дважды одной
  моделью и печатает оба ответа рядом + сводку токенов:
  - **Вариант A** — без ограничений (голый запрос).
  - **Вариант B** — system-промпт с форматом + `max_tokens` + `stop` (умный обрыв).
  - **Вариант C** — `max_tokens=20` без формата (тупой обрыв на полуслове).

Вывод C показывает разницу: `stop`/промпт — осмысленный обрыв, `max_tokens` —
гильотина по счётчику, модель про неё не знает.

## Три рычага контроля (в варианте B)

| Требование | Как сделано |
|---|---|
| Формат ответа | system-промпт: «ровно 3 пункта, каждый с `— `, ≤10 слов» |
| Длина | `max_tokens=150` + инструкция про длину пунктов |
| Завершение | `stop=["[КОНЕЦ]"]` + инструкция дописать `[КОНЕЦ]` в конце |

## Технические детали

- API: ProxyAPI, OpenAI-совместимый. base_url `https://openai.api.proxyapi.ru/v1`.
- Модель: `openai/gpt-4o-mini` (стабильно держит `stop` и `max_tokens`).
- Ключ: `PROXYAPI_KEY` из корневого `.env`.
