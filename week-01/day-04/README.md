# Неделя 1 — День 4. Температура

## Задание

Выполнить один и тот же запрос при `temperature = 0 / 0.7 / 1.2`.
Сравнить ответы по точности, креативности, разнообразию.
Сформулировать, под какие задачи годится каждая настройка.

## Формат сдачи

**Видео + код. Видео важнее кода.**

## Что делает скрипт

Один промпт (лежит в коде, легко поменять) гоняется по температурам
`[0, 0.7, 1.2, 1.5, 2.0]` на модели `gemini-2.5-flash-lite`. Два режима:

```bash
python temperature.py        # дефолтные top_p/top_k → разница идёт ТОЛЬКО от температуры
python temperature.py max    # top_p=1.0 + top_k=1000 → сняты фильтры хвоста, модель уходит в распад
```

Сравнение двух режимов при одной температуре показывает: температура поднимает
«хвост» редких токенов, а top_p/top_k решают, дотянется ли модель до мусора.

## Выводы

**Что делает температура:** регулирует разнообразие. Меняет форму распределения
вероятностей следующего слова — низкая заостряет (верхнее слово почти всегда),
высокая сглаживает (редкие слова получают шанс).

| temperature | точность | креативность | разнообразие |
|---|---|---|---|
| **0** | максимум, предсказуемо | минимум | минимум — один и тот же ответ |
| **0.7** | хорошая | средняя | средняя |
| **1.2** | падает | высокая | высокая — каждый раз другой |

**Под какие задачи:**
- **0** — факты, код, математика, классификация, перевод: один верный,
  стабильный, воспроизводимый ответ.
- **0.7** — диалог, ассистент, статьи, письма: золотая середина, дефолт.
- **1.2** — брейншторм, нейминг, слоганы, стихи, идеи: ценится непохожесть.

**Главная мысль:** температура — рычаг **«точность ↔ разнообразие»**, а НЕ
«глупее ↔ умнее». Высокая температура не делает модель умнее — делает
разнообразнее и рискованнее. Точное — вниз, творческое — вверх.

## Перевыполнение — как реально выбирается слово

Копнул механизм генерации. До итогового слова идёт конвейер (на каждое слово
заново):

```
1. температура   → задаёт форму вероятностей (поднимает/опускает хвост)
2. top_k         → выкидывает всё кроме K самых вероятных слов (нож по числу)
3. top_p         → отрезает хвост по сумме вероятностей (нож по массе; потолок-стоп)
4. renormalize   → оставшиеся пересчитываются до 100%
5. сэмплинг      → «кривая рулетка»: случайное число попадает в кусок слова → выбор
```

При `temperature = 0` рулетки нет — берётся самое вероятное слово (argmax),
отсюда детерминизм и повтор.

**Эксперимент с top_p / top_k** (режим `max`): на `temp=2.0` с дефолтными
top_p/top_k модель ещё держит форму (стих с рифмой), а сняв оба фильтра
(`top_p=1.0`, `top_k=1000`) — срывается в кашу, выдуманные слова и переключение
на английский. Граница распада — около `temp=1.5`. Вывод: температура = газ,
top_p/top_k = поводок; сводит с ума только газ + снятый поводок вместе.

Потолок температуры у Gemini — `2.0` (выше → ошибка 400). top_k нет в
OpenAI-схеме → шлём через `extra_body` (см. CLAUDE.md про квирки провайдеров).
